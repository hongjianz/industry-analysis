# 产业洞察SOP v1.0

> 基于王煜全《学会洞察行业》方法学，经过4个跨行业案例验证的行业分析框架。

## 调用规则

当用户通过 `/skill:industry-analysis` 调用时：

1. **解析参数**：
   - `--quick` 或 `-q` → 快速扫描模式（4步）
   - `--full` 或 `-f` → 完整分析模式（18步）
   - 其他文字 → 行业名称
   - 无参数 → 询问行业名称和分析目的

2. **自动判断行业类型**（分析的第一步关键决策）：

   | 特征 | → 行业类型 |
   |------|-----------|
   | 有物理产品、制造工艺、上中下游供应链 | **制造业** |
   | 纯代码/数据驱动、无物理产品、SaaS/平台 | **软件业** |
   | 未商业化、无营收、技术路线未收敛 | **前沿技术** |
   | 同时具备硬件和软件特征 | **混合型**（组合模板） |

3. **根据行业类型路由到对应模板**，见 `references/templates/`。

## --quick 快速扫描模式（4步，约1-2轮对话）

适用于快速判断一个行业值不值得深入研究。每步都产出实质性结论。

### Step 1：定义行业边界

**与用户确认：**
- 研究目的：了解趋势？投资决策？进入机会？
- 产品定义：这个行业的核心产品/服务是什么？不包括什么？

### Step 2：四维度风口判断

从技术成熟度、产业链完备性、市场潜力、竞争壁垒四个维度打分：

```
技术成熟度  ★★★☆☆  产业链完备  ★★★★☆
市场潜力    ★★★★★  竞争壁垒    ★★★☆☆

→ 总体判断：[成熟期/成长期/幼稚期]
```

**注意：不同行业类型的"成熟"标准不同：**
- 制造业：看量产良率、MARD精度等工程化指标
- 软件业：看SWE-bench得分等模型能力指标
- 前沿技术：看技术可行性（如Q>1是否验证）

### Step 3：超前指标识别

根据行业类型，找3个最重要的超前指标：

| 行业类型 | "剥洋葱"方向 | 超前指标示例 |
|---------|-------------|-------------|
| 制造业 | 上游材料/核心零部件出货量 | 硫化锂价格、酶传感器出货量 |
| 软件业 | 底层模型能力/API成本 | SWE-bench得分、模型成本下降曲线 |
| 前沿技术 | 技术里程碑+资本+政策 | Q>1验证、年度融资额、政府路线图 |

**向用户推荐3个指标，确认后使用。**

### Step 4：未来场景推演

**识别2个最不确定的关键变量**（路径分叉+不确定性+时间窗口），构建2×2矩阵：

```
                  变量2=OFF                    变量2=ON
                   ┌──────────────────┐    ┌──────────────────┐
  变量1=OFF        │ 场景A：[名称]     │    │ 场景B：[名称]     │
                   │ 概率：~XX%       │    │ 概率：~XX%       │
                   ├──────────────────┤    ├──────────────────┤
  变量1=ON         │ 场景C：[名称]     │    │ 场景D：[名称]     │
                   │ 概率：~XX%       │    │ 概率：~XX%       │
                   └──────────────────┘    └──────────────────┘
```

**用户赋概率权重后，输出核心结论。**

快速扫描结束。输出格式见 `assets/scenario-matrix.md`。

## --full 完整分析模式（5阶段18步，约3-5轮对话）

### 阶段1：定义与判断（对话交互）

| 步骤 | 内容 | 自动化程度 |
|------|------|-----------|
| ① 定义行业边界 | 研究目的+产品定义 | 🟡 引导提问，用户回答 |
| ② 行业类型判断 | → 自动路由到对应模板 | ✅ 自动 |
| ③ 技术路线分析 | 如有多条路线→多路线并行矩阵 | 🟡 自动识别，用户确认 |
| ④ 四维度风口判断 | 技术/产业链/市场/壁垒 | ✅ 自动打分，用户微调 |

### 阶段2：数据搜集（自动并行）

| 步骤 | 内容 | 自动化程度 |
|------|------|-----------|
| ⑤ 逻辑树+初始假设 | 根据已定义边界拆解并提出2-3个假设 | 🟡 Claude提出，用户确认/修正 |
| ⑥ 多维数据搜集 | WebSearch + 可用MCP工具并行搜索 | ✅ 自动（按行业类型选择数据源） |

**行业类型对应的数据源偏好：**

| 行业类型 | 优先数据源 |
|---------|-----------|
| 制造业 | 市场报告、企业财报、专利数据库、供应链新闻 |
| 软件业 | 社区数据、SWE-bench/排行榜、融资新闻、开源生态 |
| 前沿技术 | 政府政策、学术论文、融资数据、技术里程碑新闻 |

### 阶段3：分析与模型（自动推荐+人工选择）

| 步骤 | 内容 | 自动化程度 |
|------|------|-----------|
| ⑦ 超前指标识别 | 行业特定方向"剥洋葱" | 🟡 Claude推荐，用户确认 |
| ⑧ 核心模型选择 | 推荐最适合的2-3个模型 | 🟡 自动推荐，用户选择使用哪个 |

**模型选择指南：**

| 分析需求 | 推荐模型 | 适用场景 |
|---------|---------|---------|
| 宏观环境 | PEST | 任何行业的基础分析 |
| 行业阶段 | 行业生命周期 | 判断导入期→成长期→成熟期 |
| 竞争格局 | 四象限 | 2个维度可视化玩家位置 |
| 竞争关系 | 合作竞争（+互补者） | 理解"亦敌亦友"关系 |
| 公司对比 | 比较研究法 | 横向纵向对比两个实体 |
| 利润流向 | 稀缺性控制点（按行业类型） | 产业链/生态链核心分析 |

| ⑨ 生态/产业链深度分析 | 按行业类型选模板 | ✅ / 🟡 按类型自动选模板 |
| ⑩ 地缘竞争分析（可选） | 政策敏感行业单独展开 | 🟡 提醒用户是否需要 |

### 阶段4：输出与验证（人机协作）

| 步骤 | 内容 | 自动化程度 |
|------|------|-----------|
| ⑪ 金字塔结构输出 | 结论先行，逐层展开 | ✅ 自动生成，用户编辑定稿 |
| ⑫ 假设验证 | 搜索对立证据 | ✅ 自动搜集，用户判断是否被证伪 |

**假设验证是最有价值的步骤。** 不要只找支撑假设的证据，要主动搜索可能推翻假设的数据。

### 阶段5：场景与复盘

| 步骤 | 内容 | 自动化程度 |
|------|------|-----------|
| ⑬ 关键变量识别 | 路径分叉+不确定性+时间窗口 | 🟡 Claude识别，用户确认 |
| ⑭ 未来场景推演 | 2×2矩阵 | ✅ 自动生成，用户赋概率 |
| ⑮ 跟踪信号定义 | 每个场景的触发信号 | ✅ 自动生成，用户确认 |
| ⑯ 复盘迭代 | 这次SOP使用中什么有效/什么失效 | 🟡 Claude总结，用户补充 |

### 补充步骤（仅在特定行业类型执行）

| 步骤 | 仅在 | 内容 |
|------|------|------|
| ⑰ 卖铲人分析 | 前沿技术行业 | 供应链上游的真实收入信号 |
| ⑱ 开源/社区分析 | 软件业 | 社区活跃度+数据飞轮评估 |

## 使用示例

### 快速扫描
```
用户：/skill:industry-analysis --quick 固态电池

→ Claude定义边界→判断类型（制造业）→四维度评分→推荐3个超前指标→
  构建2×2矩阵→用户赋概率→输出结论
```

### 完整分析
```
用户：/skill:industry-analysis --full AI编程工具

→ Claude进入5阶段18步流程
→ 每阶段结束时汇总当前发现并问用户是否继续下一阶段
→ 最终输出完整的金字塔结构报告
```
