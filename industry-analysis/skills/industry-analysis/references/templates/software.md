# 软件业分析模板

> 当 `industry-analysis` 将行业类型判断为"软件业"时加载此模板。
> 适用场景：纯代码/数据驱动、无物理产品、SaaS/平台的行业。

## 生态链分析框架

软件行业没有物理"上中下游"，价值创造通过如下四层结构实现：

```
Layer 1：基础能力层（模型/引擎/协议）
  ├── 技术能力（模型精确度、评估指标、响应速度）
  ├── 成本结构（API调用成本、推理成本、边际成本）
  └── 护城河：训练数据+算力+人才

        ↓ API / 协议

Layer 2：平台/工具层（产品）
  ├── 核心功能（用户的"job to be done"）
  ├── 用户体验（易用性、集成度、工作流锁定）
  └── 护城河：网络效应+切换成本

        ↓ 嵌入

Layer 3：生态/社区层
  ├── 开发者社区（贡献者、插件、扩展）
  ├── 数据飞轮（更多用户→更好产品→更多用户）
  └── 护城河：社区规模+数据积累

        ↓ 触及

Layer 4：商业模式层
  ├── 定价模型（固定费/按量/BYOK/免费）
  ├── 企业级壁垒（合规、IP赔偿、审计）
  └── 财务指标（ARR、CAC、LTV、流失率）
```

## 稀缺性控制点

| 控制点 | 稀缺性 | 说明 |
|--------|-------|------|
| **🧠 模型/引擎能力** | 🔴 极高 | 技术领先度决定产品天花板 |
| **🔗 工作流锁定** | 🔴 极高 | 用户习惯+集成的切换成本 |
| **📊 数据飞轮** | 🟡 中高 | 需要用户规模启动 |
| **🏢 企业信任/合规** | 🟡 中高 | SOC2/IP赔偿/SSO等功能性壁垒 |
| **🔌 开放标准** | 🟢 降低中 | 标准化（如MCP）正在降低锁定 |

## 开源vs闭源分析

软件行业特有的竞争维度：

| 维度 | 闭源 | 开源 |
|------|------|------|
| 定价 | 订阅/按量 | 免费/BYOK |
| 护城河 | 产品体验+企业信任 | 社区+透明度+低成本 |
| 迭代速度 | 公司控制 | 社区驱动 |
| 数据隐私 | 取决于供应商政策 | 可自托管 |

**关键判断：** 开源正在从"质量较差但免费"变成"质量接近+极致低价"。

## 超前指标方向

| 类别 | 具体指标示例 |
|------|-------------|
| **底层技术** | 关键评估指标（SWE-bench等）、模型成本、推理速度 |
| **开发者社区** | GitHub Stars、贡献者增长、PR合并速度 |
| **商业指标** | ARR增速、付费用户/企业级客户数 |
| **定价模式** | BYOK采用率、价格战信号 |

## 竞争分析

- **市场份额**：按ARR/用户数/开发者占比
- **主要玩家战略**：平台型 vs 工具型 vs 生态型
- **进入壁垒**：网络效应、技术领先、开发者信任
