# 案例：AI编程工具行业

> 类型：纯软件/SaaS
> SOP模式：full（完整分析）
> 测试目的：验证SOP在"无物理供应链"行业的可迁移性

## 核心结论

> **AI编程工具正处于"代码补全→智能体"的范式转换拐点：市场$4B(2024)→$95B(2026E)，但行业结构已分化——基础模型层赢家通吃，工具层百花齐放，开源/闭源两极分化。**

## 使用的SOP步骤（含调整）

| 步骤 | 调整 | 关键产出 |
|------|------|---------|
| 行业类型判断 | 前置判断为"软件业"→改用生态链模板 | **关键路由决策** |
| 产业链分析 | ❌ 失效 | → 替换为四层价值堆栈（模型→设施→工具→应用） |
| 合作竞争模型 | 引入"开源社区"作为独特互补者 | 解释了Cursor基于VS Code开源生态建立但闭源加价的"竞合"关系 |
| 开源分析 | 新增维度 | GitHub Stars、贡献者增长、BYOK采用率 |
| 超前指标 | "上游"从物理材料→模型能力(SWE-bench得分) | 验证了方法论可迁移但方向不同 |

## 最有价值的洞察

**稀缺性控制点从"酶供应链"变成了"开发者习惯+数据飞轮+企业信任"**——三个控制点中，开发者习惯（工作流锁定）最难复制。

**定价模式的深层重构**：从$10/mo固定费(Copilot)→$20-40+用量超额(Cursor)→BYOK零加价(OpenCode)，2026年中间层加价空间被模型层挤压。
